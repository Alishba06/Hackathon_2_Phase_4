# When to Use the Frontend Agent

## Primary Use Cases

### 1. Next.js App Router Development
- When building new applications using Next.js App Router
- When migrating existing applications to App Router from Pages Router
- When restructuring existing Next.js applications to follow App Router conventions
- When implementing new features in Next.js applications using App Router
- When optimizing existing Next.js applications for better performance

### 2. Responsive UI Generation
- When creating responsive layouts that work across all device sizes
- When implementing mobile-first design approaches
- When ensuring cross-device compatibility for UI components
- When creating adaptive designs that respond to different screen sizes
- When implementing responsive design patterns using modern CSS

### 3. Performance Optimization
- When optimizing UI performance in Next.js applications
- When reducing bundle sizes and improving loading times
- When implementing code splitting and dynamic imports
- When optimizing images and media for web performance
- When minimizing unnecessary client components

### 4. Accessibility Implementation
- When implementing WCAG-compliant UI components
- When ensuring keyboard navigation support
- When adding proper ARIA attributes to components
- When creating accessible form elements and interactions
- When implementing focus management in complex UIs

### 5. Component Library Development
- When creating reusable component libraries for Next.js applications
- When implementing design systems with consistent UI patterns
- When developing themed components with dark/light mode support
- When creating accessible and well-documented UI components
- When establishing component architecture patterns

## Trigger Conditions

### Automatic Engagement
The Frontend Agent should be automatically engaged when:
- Performance metrics indicate slow loading or rendering
- Accessibility audits reveal compliance issues
- Responsive design breaks on certain screen sizes
- Bundle analysis shows oversized components
- User experience reports indicate UI/UX problems

### Manual Engagement
Manually engage the Frontend Agent when:
- Planning new UI features or components
- Reviewing existing frontend code for optimization
- Investigating UI/UX issues or bugs
- Consulting on frontend architecture decisions
- Implementing new design systems or themes

## Scenarios to Avoid Using This Agent

### 1. Backend-Related Tasks
- When working on server-side logic or APIs
- When implementing database operations
- When configuring backend infrastructure
- When working on authentication systems
- When handling business logic implementation

### 2. Non-Next.js Technologies
- When working with other frontend frameworks (Vue, Angular)
- When implementing vanilla JavaScript applications
- When working with non-React technologies
- When using different routing systems
- When working with different build tools

### 3. Infrastructure Management
- When managing cloud infrastructure directly
- When configuring web servers (Nginx, Apache)
- When handling container orchestration
- When managing networking or security at the infrastructure level
- When working on deployment pipeline configurations
